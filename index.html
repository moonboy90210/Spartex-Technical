Welcome to Spartex Technical Documentation

Developer-focused resources for streaming, broadcasting, and integration. This documentation covers RTMP/HLS/WebRTC ingest, embeddable players, the spartex.js SDK, webhooks, and production considerations.

Getting started tip: Create a developer account, generate an API key and create a channel to retrieve a stream key and start broadcasting.

Architecture Overview

Spartex runs on a horizontally scalable media pipeline designed to keep latency low while supporting adaptive bitrate streaming and global delivery via CDN. The three main layers are Input → Processing → Output.

Input Layer
  • RTMP / HLS ingest endpoints
  • WebRTC gateways for ultra-low latency
  • Stream key + token authentication
Processing Layer
  • Transcoding & adaptive bitrate ladder
  • Real-time transcoder autoscaling
  • Health checks & auto-recovery

Getting Started

Steps to start streaming with Spartex:

  1. Sign up for a developer account on the Spartex portal.
  2. Generate an API key in your profile.
  3. Create a new channel and note the channelId and stream key.
  4. Start broadcasting from your encoder (OBS, Streamlabs, vMix) or use WebRTC for near real-time.
rtmp://live.spartex.com/live
Stream Key: {your_stream_key_here}

Streaming Setup

RTMP and WebRTC are supported. RTMP for broad encoder compatibility; WebRTC for minimal latency.

Player preview placeholder
RTMP (Encoder)

Set your encoder to the Spartex RTMP ingest endpoint and your channel stream key.

WebRTC

Use the WebRTC SDK for sub-second latency interactions (chats, polls, auctions).

Example
const spartex = new SpartexWebRTC({
    apiKey: 'YOUR_API_KEY',
    channelId: 'CHANNEL_ID'
});
spartex.startBroadcast();

Player Integration

Embed Spartex Player using our CDN script and initialize with your channelId. 

<div id="spartex-player"></div>
<script src="https://cdn.spartex.com/player/v1/spartex-player.js"></script>
<script>
SpartexPlayer.init({
container: '#spartex-player',
channelId: 'CHANNEL_ID',
theme: 'dark-green',
autoplay: false
});
</script>

Spartex.js SDK

Integrate programmatically using our JavaScript SDK for channel creation, status checks, and more.

The JavaScript SDK provides simple methods to manage channels and query status.

spartex.createChannel({ name: 'Show A', privacy: 'public' })
.then(ch => console.log(ch));

Webhooks

Automate events and trigger actions when streams start, stop, or when viewers join.

Use webhooks to react to streaming events. Example events: stream.started, stream.stopped, viewer.joined.

Event Description
stream.started Fired when a broadcast goes live
stream.stopped Fired when broadcast ends
viewer.joined Fired when a viewer connects

Security

Token authentication, domain locking, DRM, and privacy controls.

Best practices: tokenized playback URLs, domain locking, DRM (when needed), and HTTPS-only delivery.

Note: Keep API keys secret — never embed them in client-side code without securing them with short-lived tokens.

Analytics

Real-time and historical metrics include viewer counts, bandwidth, device breakdowns, and geographic distribution.

Track viewer counts, watch duration, bandwidth usage, and live engagement metrics.
